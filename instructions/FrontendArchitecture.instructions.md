## Purpose
Acts as the central reference point for the structural layout, component hierarchy, client-side data management, and the overall interaction layer built for humans.

## Content Stored
- **Application Structure:** SPA vs MPA, module breakdown, lazy loading.
- **Component Hierarchy:** Core routing map, layout shells, major smart/dumb components.
- **State Management:** How data is cached, fetched, and shared (e.g., Redux, contexts).
- **Communication:** API fetching strategies, interceptors, WebSocket handling.

**Note:** When maintaining or drafting frontend structural docs, combine this with the `frontend-architecture` skill.
