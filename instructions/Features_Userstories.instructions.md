## Purpose
Stores the functional requirements, broken down into Features and individual User Stories.

## Structure & Content
**Folder Location:** `Documentations/Features/FE-XXX-[feature-name]/`

1. **Feature File (`FE-XXX-[FeatureName].md`):**
   - **Content:** High-level description of the feature, business goals, and feature-level acceptance criteria.
   - **Links:** References to related User Stories and PRD sections.

2. **User Story Files (`US-XXX-[story-name].md`):**
   - **Content:** A specific functional requirement formatted as: "As a [role], I want to [goal] so that [reason]."
   - **Acceptance Criteria:** A numbered bullet list (`[AC-001]`) of technical and functional conditions that must be met.

**Note:** To draft or structure these requirements, use the `features-and-userstories` skill.
